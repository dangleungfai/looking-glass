# Looking Glass 本地部署说明

## 一、MySQL 初始化（必做，只需一次）

在终端执行（把 `你的MySQL密码` 换成实际 root 密码；若无密码可去掉 `-p` 后的内容）：

```bash
cd /Users/dean/Documents/AI_Projects/looking-glass

# 创建数据库
mysql -u root -p你的MySQL密码 -e "CREATE DATABASE IF NOT EXISTS looking_glass CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;"

# 建表
mysql -u root -p你的MySQL密码 looking_glass < backend/db/schema.sql

# 初始数据（角色、示例 POP/设备、命令模板等）
mysql -u root -p你的MySQL密码 looking_glass < backend/db/seed.sql

# 若希望用 root 直连，可跳过创建 lg_user，并修改 backend/src/main/resources/application.yml 中
# spring.datasource.username 为 root、password 为你的 MySQL root 密码。
# 否则创建应用账号（与 application.yml 默认一致）：
# mysql -u root -p你的MySQL密码 -e "CREATE USER IF NOT EXISTS 'lg_user'@'localhost' IDENTIFIED BY 'lg_password'; GRANT ALL ON looking_glass.* TO 'lg_user'@'localhost'; FLUSH PRIVILEGES;"
```

## 二、启动三个服务

在项目根目录下打开 **3 个终端**，分别执行：

### 终端 1 - 后端（端口 8080）

```bash
cd /Users/dean/Documents/AI_Projects/looking-glass/backend
export JAVA_HOME=/opt/homebrew/opt/openjdk@17   # 如未装 OpenJDK@17 可改为你本机 Java 路径
java -jar build/libs/looking-glass-backend-0.0.1-SNAPSHOT.jar
```

### 终端 2 - Worker（端口 8000）

```bash
cd /Users/dean/Documents/AI_Projects/looking-glass/worker
source .venv/bin/activate
uvicorn main:app --host 0.0.0.0 --port 8000
```

### 终端 3 - 前端（端口 3000）

```bash
cd /Users/dean/Documents/AI_Projects/looking-glass/frontend
npm run dev
```

## 三、访问

- **公网首页 / 查询**: http://localhost:3000  
- **后台管理**: http://localhost:3000/admin  
- **默认管理员**: 用户名 `admin`，密码 `admin123`（首次启动后端时自动创建）

## 四、可选：后台一键启动

若已完成「一、MySQL 初始化」，可在项目根目录执行：

```bash
./scripts/start-all.sh
```

会依次在后台启动后端、Worker、前端，日志在 `logs/` 目录。停止时执行：

```bash
./scripts/stop-all.sh
```

## 故障排查

- **后端启动报错 “Access denied” 或 “Unknown database”**：检查 MySQL 已执行步骤一，且 `backend/src/main/resources/application.yml` 里用户名密码为 `lg_user` / `lg_password`（与 seed 中一致；若你改了 MySQL 用户，需同步改配置）。
- **前端请求 API 404**：确认后端已启动在 8080，且前端通过 Vite 代理访问（`npm run dev` 已配置 proxy 到 8080）。
- **查询执行失败**：Worker 需能 SSH 到设备；当前为示例配置，若没有真实设备，可先在后台添加测试用 POP/设备，或查看 Worker 日志排查连接错误。
